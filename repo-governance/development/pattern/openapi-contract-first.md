# OpenAPI Contract-First Development

Contract-first means the OpenAPI specification is the source of truth for all API
contracts. Types and clients are generated from the spec, not the other way around.

See [Hexagonal Architecture — Backend Apps](./hexagonal-architecture-be.md) for the
bounded-context structure that hosts the generated contract types.

## Overview

In contract-first development:

1. The OpenAPI spec is authored and reviewed before any implementation begins.
2. Typed clients and server stubs are generated from the spec automatically.
3. Any drift between implementation and spec is detected by re-running codegen and
   checking `git diff`.

This eliminates a class of bugs where frontend and backend teams diverge on field names,
types, or status codes.

## Single Source of Truth

The canonical OpenAPI specification for the CRUD demo lives at:

```
specs/apps/crud/containers/contracts/openapi.yaml
```

This file is the authoritative source. Do not hand-edit generated artifacts; regenerate
them from this file instead.

## Bundled Artifact

The bundled artifact is generated by the `crud-contracts:bundle` Nx target and lands at:

```
specs/apps/crud/containers/contracts/generated/openapi-bundled.yaml
```

Run bundling with:

```bash
npx nx run crud-contracts:bundle
```

The bundled file resolves all `$ref` references into a single self-contained YAML document.
All codegen targets consume the bundled artifact, not the source spec directly.

language-specific contract types into `generated-contracts/` inside the app.

## Frontend Consumer Codegen

The three primary FE apps also consume the bundled spec to generate typed API clients:

| App                         | Codegen command                                |
| --------------------------- | ---------------------------------------------- |
| `crud-fe-ts-nextjs`         | `npx nx run crud-fe-ts-nextjs:codegen`         |
| `crud-fe-ts-tanstack-start` | `npx nx run crud-fe-ts-tanstack-start:codegen` |
| `crud-fe-dart-flutterweb`   | `npx nx run crud-fe-dart-flutterweb:codegen`   |

Generated client artifacts land in `generated-contracts/` inside each FE app's source tree.

## Drift Detection

To detect contract drift, re-run all codegen targets and check whether the generated
artifacts changed:

```bash
npx nx run-many -t codegen
git diff --quiet generated-contracts/
```

If `git diff` exits non-zero, the spec has drifted from the last generated artifacts.
Commit the updated artifacts or fix the spec before merging.

## Principles Implemented/Respected

- **[Explicit Over Implicit](../../principles/software-engineering/explicit-over-implicit.md)** —
  The OpenAPI spec is the single explicit source of truth; generated types prevent implicit drift
  between frontend and backend teams.
- **[Automation Over Manual](../../principles/software-engineering/automation-over-manual.md)** —
  Codegen targets in each app automatically produce typed clients and server stubs from the spec,
  replacing manual type maintenance.
- **[Root Cause Orientation](../../principles/general/root-cause-orientation.md)** — Drift
  detection via `git diff` on generated artifacts surfaces contract mismatches at their root (the
  spec) rather than through runtime failures.

## Conventions Implemented/Respected

- **[Hexagonal Architecture — Backend Apps](./hexagonal-architecture-be.md)** — Generated
  contract types land in the adapter zone (`generated-contracts/`) inside each BE app, conforming
  to the bounded-context structure.
